# NMI Agent Project

This folder contains the local agents for forecastability review, score tuning, and the final closed-loop validation run.

```text
nmi_agent_project/
  cv_review_agent/
    nmi_classification_cv_agent.py
    nmi_classification_cv_agent_api_key.txt
    requirements.txt
    README.md
    outputs/
  score_tuning_agent/
    project_planner_agent.py
    PROJECT_PLAN.md
    score_tuning_agent.py
    config/
    prompts/
    skills/
    outputs/
  final_test/
    final_test_driver.py
    config/
      start_config_random.json
      final_scoring_config.json
    README.md
    outputs/
```

Run order:

1. Generate the threshold-based classification table with `EDA_Tasks/nmi_score_classification_threshold.py`.
2. Run `cv_review_agent/nmi_classification_cv_agent.py`.
3. Run `score_tuning_agent/score_tuning_agent.py`.
4. Run the final closed-loop test:

```powershell
python nmi_agent_project\final_test\final_test_driver.py
```

The final test defaults are `--min-loops 2`, `--max-loops 5`, and `--random-trials 10000`. The driver starts from `final_test/config/start_config_random.json`, runs full CV review and random tuning repeatedly, and forces at least two complete rounds before it can stop. The last accepted run was `forced_two_rounds_10000_20260513_024713`: it completed 2 rounds, converged on round 2, and its stable selected config is copied to `final_test/config/final_scoring_config.json`.

Generated `outputs/` folders and API keys are local artifacts, not source files.

One-step full run:

```powershell
python nmi_agent_project\main.py
```

Override the final-test loop bounds and output directory:

```powershell
python nmi_agent_project\main.py `
  --min-run 2 `
  --max-run 5 `
  --output nmi_agent_project\final_test\outputs\my_run
```

`main.py` runs the four steps above in order. The final-test stage uses the random score-tuning config generated by step 3 as its `--start-config`.
